## 2. SÉCURITÉ — CRITIQUE 🚨

### Threat Model: Kit distribué publiquement

Quand un kit est **public + pré-configuré** avec hooks/settings, l'attaque surface se dilate:
1. User clone le repo
2. Hooks auto-actifs → exécutent du bash sans vérification de signature
3. settings.json pré-rempli → si compromis → privilege escalation

### Findings spécifiques

**CRITIQUE: settings.json privilege escalation**
- Claude Code v2.1.1 vulnerability (patched v2.1.2): Si settings.json missing, malicious code peut le créer avec hooks injectés
- **Impact:** Code runs avec user privileges au prochain Claude Code restart
- **Risk level:** *Haute* si repo fork + attacker ajoute malicious hook
- **Attestation:** Anthropic security bulletin + GitHub issues

**HAUT: Bash hooks without signature verification**
- Hooks actuels: `no-env-guard.sh`, `pre-compact.sh`, etc. — pas signés
- **Attaque:** Fork repo + modify `pre-compact.sh` → code exécuté à chaque session
- **Mitigation existante:** git + checksum validation (faible)

**HAUT: Shell RC pollution**
- Si user ~.bashrc/~.zshrc output du texte (welcome messages, fortune, etc.)
- → JSON parsing de hook output se casse
- **Impact:** Security checks silently fail, no error visible

**MOYEN: Secrets in .env but not protected in git**
- Kit inclut `.env.example` — users vont copier → risque de committer credentials
- **Mitigation:** `.env` dans `.gitignore` (OK)
- **Gap:** Pas de pre-commit hook qui refuse les patterns (API_KEY, SUPABASE_, etc.)

### Recommandations

**Avant distribution v1.0:**

1. ✅ **Add file integrity check**
   ```bash
   # Dans onboarding.sh après clone
   sha256sum -c .claude/hooks.sha256 || exit 1
   ```
   - Générer SHA256 de chaque hook, commit dans repo
   - User peut vérifier que hooks n'ont pas été modifiés

2. ✅ **Add pre-commit hook refuse secrets**
   ```bash
   # ~/.claude/hooks/pre-commit-secrets.sh
   if git diff --cached | grep -E 'SUPABASE_KEY|API_KEY|sk_live_|sk_test_'; then
     echo "ERROR: Secrets detected in commit"
     exit 1
   fi
   ```

3. ✅ **Audit no-env-guard.sh** — confirmer qu'il catch bien les secrets patterns
   - Actuellement: regex check sur Write tool use
   - À vérifier: edge cases (base64 encoded, .env files being read)

4. ✅ **Add settings.json backup + readonly on install**
   ```bash
   # Dans onboarding.sh
   chmod 600 ~/.claude/settings.json
   cp ~/.claude/settings.json ~/.claude/settings.json.backup
   ```

5. ✅ **Documenter** dans SECURITY.md (NOUVEAU)
   - Risques des kits publics
   - Conseils pour vérifier l'intégrité après clone
   - Où signaler les security issues (Responsable Disclosure)

## 4. TOKEN COST & SCALING — CRITIQUE 💰

### Current pricing (March 2026)

- **Opus 4.6:** $5/M tokens input, $25/M output
- **Sonnet 4.6:** $3/M input, $15/M output
- **Haiku 4.5:** $1/M input, $5/M output
- **Average cost per dev:** $100-200/month (all models)

### Focus Kit token consumption analysis

Kit uses:
- **9 core agents** (ghost=Opus, 8=Sonnet)
- **30+ hook invocations** per session on average
- **3-5 hooks per user prompt** (pre-check, post-action, validation)
- **Agent Teams** enabled by default (7x token multiplier vs single session)

### Scaling scenario: 100 users

**Conservative estimate per user:**
- 1 hour/day Claude usage (typical: dev/designer/CEO)
- 4 agent calls/hour average
- Hook overhead: +20% tokens

**Calculation:**
- Single session (Sonnet): ~150K tokens/hour
- With 9 agents (distributed): ~600K tokens/hour (4 simultaneous)
- With hooks (pre+post): ~720K tokens/hour
- At $15/M output: ~$10.80/hour
- Per day (1h): ~$11/day
- Per month (22 work days): **~$242/month per user**

**100 users = $24,200/month**

### Where cost explodes

1. **Agent Teams at scale:** If 10+ agents run simultaneously (big projects)
   - Cost multiplier 10x single session
   - 1 project = $100+ in tokens

2. **Vision360 with web search:** Deep research = 50K+ tokens
   - /vision360 command might cost $5-10 per invocation

3. **Hooks running every prompt:** 30 hooks × 3K tokens each = 90K token overhead per session
   - **This is a hidden cost that users won't see until first bill**

### User expectation gap

- **User expects:** "Kit is free/open-source → no cost"
- **Reality:** "Kit is free/open-source → but Claude API calls cost $200+/month"
- **Risk:** Users blame kit for high bills, switch to competitors

### Recommendations

✅ **Add cost transparency to CLAUDE.md**
```markdown
## Token Cost Estimation

Focus Kit uses agent teams by default. Monthly cost depends on usage:
- Light (5h/week): ~$50/month
- Medium (15h/week): ~$150/month
- Heavy (40h/week): ~$400+/month

Optimize: Use Haiku for repetitive tasks, Opus only for planning.
```

✅ **Add Batch API docs for cost reduction**
- Batch API = 50% discount on tokens
- Great for async workflows, daily reports, etc.
- Link to Anthropic guide

✅ **Add token monitoring hook**
```bash
# New hook: PostSession → logs token usage to MEMORY.md
# Users can track spending vs productivity
```

✅ **Warn about Agent Teams cost**
- Document in onboarding-workflow.yaml
- "Using 5 agents simultaneously = 5x token cost"
- Recommend "start with 3, add more as needed"

    done

## 9. SCALABILITY & MULTI-USER — MOYEN 🔧

### Potential issues at 100-1000 users

**Settings.json conflicts:**
- If multiple agents modify settings.json simultaneously (unlikely)
- → JSON corruption possible
- **Mitigation:** Atomic writes (already handled by Claude Code)

**Agent name collisions:**
- User creates agent named "tate" (custom backend agent) → conflicts with core
- → Confusion, broken workflows
- **Mitigation:** Namespace custom agents: `tate-custom-[user]`

**MCP server resource contention:**
- Playwright MCP × 100 users = 100 browser processes?
- → Each user isolated, no sharing
- **Risk:** Host resources if run on shared machine
- **Mitigation:** Document resource requirements

**Shared knowledge base (if team deployment):**
- Lafouine indexing per-user only currently
- If team wants shared knowledge → gap
- **Future:** Enterprise version with team knowledge

### Recommendations

✅ **Add ARCHITECTURE.md** to document assumptions

```markdown
## Single-user assumption

Focus Kit is designed for individual developers/creators.
- Agent state: per-session (not persisted across users)
- Knowledge: per-user (Lafouine indexes locally)
- Agent naming: global (prevent collisions with namespacing)

For team deployment:
- Each user gets own kit clone
- Share CLAUDE.md rules via repo fork (not agent state)
- Knowledge sharing: manual export/import via Lafouine
```

✅ **Add agent naming guidelines**
- Reserve `ghost, dre, tate, tasha, light, storm, tommy, forge, lafouine`
- User custom agents: `[domain]-[function]` format
- Example: `legal-contracts`, `marketing-ads`, `science-research`
