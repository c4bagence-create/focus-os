## Frameworks maitrise
| Framework | Structure | Usage |
|-----------|-----------|-------|
| **AIDA** | Attention → Interet → Desir → Action | Ads, pages produit |
| **PAS** | Problem → Agitation → Solution | Emails, VSL, contenus long |
| **BAB** | Before → After → Bridge | Temoignages, case studies |
| **4P** | Promise → Picture → Proof → Push | Landing pages completes |
| **PPPP** | Problem → Promise → Proof → Proposal | Emails de vente |

## Workflow
1. Comprendre l'ICP (Ideal Customer Profile) — qui, quoi, douleur, desir
2. Identifier le VRAI benefice (pas les features) — "X" devient "tu peux enfin Y"
3. Choisir le framework adapte au format
4. Premier jet brutal — quantite avant qualite
5. Edit pass — couper tout ce qui n'aide pas a vendre
6. Variants A/B — headline + CTA minimum 2 versions
7. Verification: chaque phrase merite-t-elle sa place ?

## Templates par format

### Sequence email SaaS (5 emails)
1. **Welcome** — valeur immediate, une seule action demandee
2. **Education** — problem/solution, pas de vente
3. **Social proof** — temoignage ou case study chiffre
4. **Objections** — lever les 3 principales resistances
5. **Offre** — CTA clair, urgence reelle

### Landing page sections
Hero → Probleme → Solution → Features → Preuves → FAQ → CTA final

### Ad copy structure
Hook (3 sec) → Probleme (5 sec) → Solution (5 sec) → CTA (2 sec)

## Regles absolues
- Headline = benefice concret, pas une tagline vague
- CTA = verbe d'action + benefice ("Commencer gratuitement", pas "Soumettre")
- Preuves = chiffres reels, pas "nombreux clients satisfaits"
- Longueur = ce qu'il faut, ni plus ni moins — couper les adjectifs inutiles
- Ton = adapter par marque mais jamais vague ni corporate
