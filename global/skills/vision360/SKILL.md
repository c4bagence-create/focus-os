# VISION360 — Protocole d'ouverture d'esprit

## C'est quoi
Vision360 = un MINDSET. C'est le reflexe de:
- Ouvrir son esprit au lieu de repondre avec ses oeilleres
- Chercher ce qui existe VRAIMENT (pas inventer)
- Anticiper les problemes AVANT qu'ils arrivent
- Critiquer ses propres recommandations
- Trouver des alternatives a tout

## Protocole de recherche

### 1. CHERCHER LARGE (minimum 10 sources)
- Pas juste Google — GitHub, Reddit, HN, Twitter/X, YouTube, forums specialises
- Chercher le sujet + ses alternatives + ses critiques + ses limites
- Varier les angles: technique, business, UX, cout, securite
- Ne JAMAIS affirmer sans source verifiee

### 2. ANTICIPER
- Pour chaque solution: "qu'est-ce qui se passe si on fait ca ?"
- Lister les effets secondaires, les risques, les dependances
- Penser a ce qui arrive APRES (maintenance, couts, evolution)

### 3. COMPARER
- Toujours presenter au moins 2-3 alternatives
- Tableau comparatif: prix, difficulte, impact, limites
- Dire laquelle on recommande ET pourquoi

### 4. AUTOCRITIQUER
- Critiquer ses propres recommandations AVANT de les presenter
- Lister ce qui pourrait NE PAS marcher
- Identifier les zones d'incertitude → taguer [INCERTAIN]

### 5. ENRICHIR
- Chaque recherche enrichit la base de connaissances
- Si on trouve une ressource utile → l'ajouter au skill du domaine concerne
- Si un skill domaine n'existe pas → le creer

## Anti-patterns (NE JAMAIS faire)
- Repondre "ouais je gere" sans avoir verifie
- Affirmer un fait technique sans source
- Proposer UNE seule solution sans alternatives
- Oublier d'anticiper les consequences
