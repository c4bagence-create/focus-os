# PROJECT TYPE: Presentation

> Tu veux une presentation ? Voici le chemin COMPLET.
> Pas de zigzag — chaque etape produit un livrable, chaque gate valide avant de passer a la suite.
> Duree estimee: 1-2 heures (Quick Flow) / 1 jour (Full)

---

## Vue d'ensemble

```
ETAPE 1: Brief                  → Objectif + audience + contraintes
  ↓ GATE 1: Brief valide
ETAPE 2: Structure slides       → Storyboard de la presentation
  ↓ GATE 2: Structure validee
ETAPE 3: Contenu                → Textes + donnees + arguments
  ↓ GATE 3: Contenu complet
ETAPE 4: Design DA              → Direction artistique des slides
  ↓ GATE 4: Design valide
ETAPE 5: Animations             → Transitions et micro-interactions
  ↓ GATE 5: Animations fluides
ETAPE 6: Review                 → Relecture complete
  ↓ GATE 6: Review OK
ETAPE 7: Export                 → Livrable final
```

**Resultat final:** Presentation HTML interactive ou PDF premium, prete a presenter.

## ETAPE 1 — Brief

**Pourquoi cette etape:** Une presentation sans objectif clair c'est du PowerPoint deco. Le brief force a definir POURQUOI tu presentes, a QUI, et ce que tu veux qu'ils FASSENT apres. Ca change tout: le ton, la longueur, les visuels, le niveau de detail.

**Agent:** ghost (strategie)
**Duree:** 10 min

### Ce qu'on fait
1. **Objectif:** Qu'est-ce que l'audience doit FAIRE apres la presentation ?
   - Acheter / investir / valider un projet / apprendre / etre convaincu
2. **Audience:**
   - Qui: clients, investisseurs, equipe interne, apprenants, partenaires
   - Niveau: expert ou debutant sur le sujet
   - Attentes: qu'est-ce qu'ils esperent voir/entendre ?
3. **Contexte:**
   - Live (sur scene, reunion) ou envoye par email (lecture solo)
   - Duree: 5 min pitch / 20 min standard / 45 min formation
   - Avec ou sans speaker (la pres parle d'elle-meme ou accompagne un discours)
4. **Format:**
   - HTML interactif (navigation clavier, animations, responsive)
   - PDF (pour envoi email, impression)
   - Les deux (HTML pour presenter, PDF pour envoyer apres)
5. **Style:**
   - Dark premium / Light corporate / Colore energique
   - Reference visuelle: 1-2 presentations d'inspiration

### Livrable
`{projet}/_focus-output/presentation/brief.md`

### GATE 1 — Brief valide
- [ ] Objectif en 1 phrase (l'audience doit faire quoi apres ?)
- [ ] Audience definie
- [ ] Duree et contexte determines
- [ ] Format choisi (HTML/PDF/les deux)
- [ ] Brief valide

## ETAPE 2 — Structure slides

**Pourquoi cette etape:** Une bonne presentation raconte une HISTOIRE. Les slides ne sont pas des documents — ce sont des etapes d'un voyage narratif. La structure definit le rythme, les transitions, les moments forts. Sans elle, tu empiles des infos sans fil conducteur.

**Agent:** ghost (structure narrative)
**Skills a charger:** `copywriting` (structures narratives)
**Duree:** 15-20 min

### Ce qu'on fait
1. **Arc narratif — choisir la structure:**

   **Structure probleme-solution (la plus courante):**
   - Slide 1: Accroche (stat choc, question provocante, image forte)
   - Slides 2-3: Le probleme (ce que l'audience vit)
   - Slides 4-5: Pourquoi ca existe (context, causes)
   - Slides 6-8: La solution (notre proposition)
   - Slides 9-10: La preuve (chiffres, temoignages, demo)
   - Slide 11: L'appel a l'action (ce qu'on veut qu'ils fassent)

   **Structure pitch (investisseurs, clients):**
   - Le marche / Le probleme / Notre solution / Comment ca marche / Traction / Equipe / Ask

   **Structure formation (apprentissage):**
   - Hook / Contexte / Concept 1 / Exemple / Concept 2 / Exemple / Recapitulatif / Action

2. **Storyboard:**
   - Pour chaque slide: titre + message cle (1 phrase) + type de visuel
   - 1 slide = 1 idee (PAS 2, PAS 3 — UNE seule)
   - Compter: max 1-2 min par slide en live
3. **Moments forts:**
   - La "slide surprise" (stat, visuel, revelation)
   - Les transitions (comment on passe d'une section a l'autre)
4. **Nombre de slides:**
   - Pitch 5 min: 8-12 slides
   - Standard 20 min: 15-25 slides
   - Formation 45 min: 30-50 slides

### Livrable
`{projet}/_focus-output/presentation/structure.md`

### GATE 2 — Structure validee
- [ ] Arc narratif choisi et justifie
- [ ] Chaque slide a un titre + message cle
- [ ] 1 slide = 1 idee
- [ ] Moments forts identifies
- [ ] Nombre de slides coherent avec la duree
- [ ] Structure validee

## ETAPE 3 — Contenu

**Pourquoi cette etape:** La structure donne le squelette, le contenu donne la chair. C'est le moment de rediger les textes, trouver les chiffres, formuler les arguments. Attention: sur un slide, MOINS c'est MIEUX. Si tu dois lire ton slide, ta presentation est ratee.

**Agent:** tasha (copy), ghost (strategie), storm (recherche donnees)
**Skills a charger:** `copywriting` (hooks, formulation impactante), `vision360` (si besoin de donnees/stats)
**Duree:** 20-30 min

### Ce qu'on fait
1. **Textes par slide:**
   - Titre: percutant, court (max 7 mots)
   - Body: 3-5 bullet points MAX OU 1 phrase forte
   - Regle 6x6: max 6 bullets, max 6 mots par bullet (guide, pas loi)
   - Notes speaker (si presentation live): ce qu'on DIT a l'oral
2. **Donnees et chiffres:**
   - Sources verifiees pour chaque stat
   - Visuels pour les chiffres (graphiques, barres, comparaisons)
   - Arrondir pour la lisibilite (73.2% → 73%)
3. **Arguments:**
   - Chaque affirmation est supportee par une preuve
   - Temoignages avec nom + contexte (pas "un client a dit...")
   - Comparaisons parlantes ("X fois plus rapide que Y")
4. **CTA slide final:**
   - Clair, specifique, avec les next steps
   - Contact / lien / QR code si pertinent

### Livrable
Contenu textuel pour chaque slide

### GATE 3 — Contenu complet
- [ ] Chaque slide a son texte final
- [ ] Stats sourcees
- [ ] Max 5 bullet points par slide
- [ ] CTA final clair
- [ ] Ton coherent du debut a la fin
- [ ] Contenu valide

## ETAPE 4 — Design DA

**Pourquoi cette etape:** Le design fait 50% de l'impact d'une presentation. Un contenu genial sur des slides Word Art, personne ne le prend au serieux. La DA (direction artistique) definit la palette, la typo, la mise en page — tout ce qui fait que ta pres a l'air pro et premium.

**Agent:** dre (design), tommy (DA)
**Duree:** 30-45 min

### Ce qu'on fait
1. **Design system de la presentation:**
   - Palette: 2-3 couleurs max (fond, accent, texte)
   - Typographie: 1 font max (2 si vraiment necessaire)
   - Grid: alignement des elements (pas de placement "au feeling")
   - Spacings: marges consistantes
2. **Types de slides:**
   - **Title slide**: grand titre, minimaliste
   - **Content slide**: titre + body + visuel
   - **Stat slide**: 1 gros chiffre, 1 phrase d'explication
   - **Quote slide**: temoignage mis en valeur
   - **Image slide**: full-bleed visuel avec overlay texte
   - **Comparison slide**: cote a cote ou avant/apres
   - **CTA slide**: action claire, contact, next steps
3. **Visuels:**
   - Images haute qualite (pas de stock photos generiques)
   - Icones si necessaire (style uniforme: outline OU filled, pas les deux)
   - Graphiques clean (pas de 3D, pas de gradient arc-en-ciel)
4. **Mise en page:**
   - Respirer: espace blanc = respiration visuelle
   - Hierarchie: le plus important = le plus gros
   - Coherence: meme mise en page pour le meme type de contenu

### Livrable
Slides designees (HTML ou fichier design)

### GATE 4 — Design valide
- [ ] Design system defini (palette + typo + grid)
- [ ] Tous les slides designes
- [ ] Coherence visuelle de la premiere a la derniere slide
- [ ] Lisible (contraste, taille du texte)
- [ ] Premium (pas de clip art, pas de stock photos cheap)
- [ ] Design valide

## ETAPE 5 — Animations

**Pourquoi cette etape:** Les animations donnent du RYTHME a la presentation. Elles guident l'oeil, creent de la surprise, et maintiennent l'attention. Mais trop d'animations = distraction. L'objectif c'est subtilite et fluidite, pas feu d'artifice.

**Agent:** dre (animations frontend)
**Duree:** 15-20 min

### Ce qu'on fait
1. **Transitions entre slides:**
   - Fade (le plus sur, clean)
   - Slide horizontal (style keynote)
   - Pas de transition differente a chaque slide (coherence)
2. **Animations intra-slide:**
   - Elements qui apparaissent progressivement (texte, bullet points)
   - Chiffres qui comptent (counter animation)
   - Images qui fade-in
   - Barres de progression qui se remplissent
3. **Interactions (HTML uniquement):**
   - Navigation clavier (fleches gauche/droite)
   - Progress bar en bas
   - Indicateur de slide (3/25)
   - Fullscreen toggle
4. **Regles:**
   - Duree des animations: 300-500ms (pas plus lent, ca endort)
   - Easing: ease-out ou cubic-bezier (pas de linear)
   - JAMAIS de son automatique
   - JAMAIS de GIF anime (sauf si c'est volontairement humoristique)

### Livrable
Animations integrees dans les slides

### GATE 5 — Animations fluides
- [ ] Transitions coherentes entre toutes les slides
- [ ] Animations subtiles (pas de surcharge)
- [ ] Navigation clavier fonctionnelle (si HTML)
- [ ] Pas de lag/saccade
- [ ] L'animation SERT le contenu (pas l'inverse)

## ETAPE 6 — Review

**Pourquoi cette etape:** Avant de presenter, quelqu'un d'AUTRE doit voir la pres. Les fautes, les incoherences, les slides confuses — l'auteur ne les voit plus. La review c'est le filet de securite.

**Agent:** light (QA), ghost (contenu)
**Duree:** 10-15 min

### Ce qu'on fait
1. **Contenu:**
   - Zero faute d'orthographe/grammaire
   - Chiffres corrects et sources verifiables
   - Coherence du fil narratif (pas de slide orpheline)
   - CTA final clair et actionnable
2. **Design:**
   - Alignement des elements
   - Taille du texte lisible (meme au fond de la salle)
   - Contraste suffisant
   - Images non pixelisees
3. **Technique (HTML):**
   - Toutes les animations fonctionnent
   - Navigation clavier sans bug
   - Responsive (si partage ecran de taille differente)
   - Fonctionne sur Chrome + Safari
4. **Timing:**
   - Faire un run-through complet
   - Verifier que le nombre de slides correspond a la duree
5. **Test audience:**
   - Montrer a 1-2 personnes non impliquees
   - "Est-ce que tu comprends le message principal ?"

### Livrable
`{projet}/_focus-output/presentation/review.md` (corrections identifiees)

### GATE 6 — Review OK
- [ ] Zero faute
- [ ] Coherence narrative validee
- [ ] Design propre et aligne
- [ ] Technique sans bug
- [ ] Run-through fait dans le temps imparti

## ETAPE 7 — Export

**Pourquoi cette etape:** L'export c'est le packaging final. Selon le contexte (live, email, archivage), le format change. Et il faut s'assurer que la presentation fonctionne PARTOUT — pas juste sur ton ecran.

**Agent:** dre (export technique)
**Duree:** 10-15 min

### Ce qu'on fait
1. **Format HTML interactif:**
   - Fichier standalone (tout inline — CSS, JS, images en base64)
   - OU dossier avec assets (si images lourdes)
   - Navigation clavier + boutons
   - Responsive
   - Deploy Vercel si besoin d'URL partageable
2. **Format PDF:**
   - Export depuis le HTML (Playwright → PDF)
   - OU creation directe PDF
   - 1 slide = 1 page
   - Embedder les fonts
   - Taille: A4 landscape ou 16:9
3. **Backup:**
   - URL live (si HTML)
   - Fichier PDF en local
   - Sources (HTML + assets) archivees

### Livrable
Presentation finale en HTML et/ou PDF

### Gate finale — Presentation livree
- [ ] Format choisi exporte et fonctionnel
- [ ] URL partageable (si HTML)
- [ ] PDF genere (si demande)
- [ ] Sources archivees
- [ ] Run-through final valide

## Quick Flow (version rapide)

Pour une presentation express en 1h.

```
QUICK-BRIEF (5 min) → QUICK-STRUCTURE (10 min) → QUICK-BUILD (30 min) → QUICK-EXPORT (10 min)
```

### Quick-Brief
- Objectif + audience + duree en 3 lignes
- 1 reference d'inspiration

### Quick-Structure
- Liste des slides avec titre + message cle
- Arc narratif simple (probleme → solution → action)

### Quick-Build
- dre code les slides directement en HTML
- Design simple: fond sombre, texte blanc, 1 accent
- Animations minimales (fade uniquement)

### Quick-Export
- Fichier HTML standalone
- Deploy Vercel si besoin d'URL
