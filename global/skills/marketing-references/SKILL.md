## 1. FRAMEWORKS DE PERSUASION

### Choisir selon le niveau de conscience du prospect
| Awareness level | Framework | Usage |
|----------------|-----------|-------|
| Pas conscient du probleme | AIDA | Ads froides, contenu educatif, top of funnel |
| Conscient du probleme | PAS | Service pages, landing pages, ads ciblees |
| Cherche deja une solution | BAB | Sales pages, case studies, temoignages |
| Pret a acheter | 4P / Close | Pages de vente longues, emails finaux |

### AIDA — Attention, Interest, Desire, Action
- **A** : Hook qui stoppe le scroll (stat choc, question, contradiction)
- **I** : Relier a la vie/business du lecteur (benefice concret)
- **D** : Preuve sociale, transformation visible, urgence emotionnelle
- **A** : CTA unique, precis, positionne apres la valeur

### PAS — Problem, Agitate, Solution
- **P** : Nommer le probleme exactement (le prospect doit se reconnaitre)
- **A** : Aggraver, amplifier le cout de ne PAS resoudre
- **S** : Offrir le chemin de sortie = ton produit

### BAB — Before, After, Bridge
- **Before** : Situation actuelle douloureuse
- **After** : Vision de la transformation
- **Bridge** : Comment tu y arrives (ton offre)

### 4P — Promise, Picture, Proof, Push
- **Promise** : La transformation principale en 1 phrase
- **Picture** : Visualiser la vie apres (storytelling)
- **Proof** : Temoignages, stats, cas clients
- **Push** : CTA + urgence + garantie

### StoryBrand (Donald Miller)
- **Character** : Le client est le heros, pas la marque
- **Problem** : Externe (visible) + Interne (emotionnel) + Philosophique
- **Guide** : La marque est le guide (empathie + competence)
- **Plan** : 3 etapes simples
- **Action** : CTA direct + CTA transitoire
- **Success/Failure** : Montrer les 2 scenarios

### Hybridation (niveau expert)
PAS pour le hook → FAB (Features, Advantages, Benefits) pour le corps → AIDA pour le close.

|-------|-------------------|--------------|-----------|

| SEO | CTR position 1 | 25-30% | 40%+ |

| SaaS | LTV/CAC | > 3:1 | > 5:1 |

### Metriques SaaS critiques
- **MRR** (Monthly Recurring Revenue) — sante globale
- **ARR** (Annual Run Rate) — MRR x 12
- **Churn** — % clients perdus par mois (< 5% = viable)
- **NRR** (Net Revenue Retention) — > 100% = expansion > churn
- **CAC** (Customer Acquisition Cost) — tout cout / nb clients acquis
- **LTV** (Lifetime Value) — ARPU / Churn rate mensuel
- **Payback period** — CAC / (ARPU x marge brute) — cible < 12 mois
- **DAU/MAU** — engagement quotidien vs mensuel (> 20% = bon)

## 4. TEMPLATES EMAIL

### Structure email haute performance
1. **Sujet** : Curiosite + specifique + < 50 caracteres
2. **Preview text** : Complement du sujet, pas repetition
3. **Ouverture** : Hook 1 phrase (question, stat, declaration choc)
4. **Corps** : 1 seul angle, paragraphes courts (2-3 lignes max)
5. **CTA** : 1 seul bouton, benefice formule comme action
6. **PS** : Rappel offre ou CTA alternatif (souvent le plus lu)

### Sequences par objectif
| Type | J0 | J1 | J3 | J7 | J14 | J30 |
|------|----|----|----|----|-----|-----|
| **Welcome SaaS** | Bienvenue + quick win | Feature cle #1 | Cas d'usage client | Upgrade trigger | -- | Re-engagement |
| **E-commerce** | Confirmation commande | Upsell produits complementaires | Tuto / tips usage | Review request | -- | -- |
| **Abandon panier** | Rappel panier (1h) | Social proof (24h) | Urgence/reduction (48h) | -- | -- | -- |
| **Re-engagement** | -- | -- | -- | -- | -- | "Tu nous manques" + incentive |
| **Post-achat** | Confirmation | Tips utilisation | Cross-sell | Review | -- | Reachat |

### Sujets haute performance (formules)
- `[Prenom], tu as oublie quelque chose` (abandon panier)
- `Pourquoi [CONCURRENT] a perdu 40% de clients` (curiosite)
- `3 erreurs que 90% des [METIER] font` (education)
- `[Prenom] — voici ton acces exclusif` (urgence/exclusivite)
- `Ce que personne ne te dit sur [SUJET]` (open loop)

## 6. LANDING PAGE — STRUCTURE OPTIMISEE

### Structure haute conversion (ordre valide)
1. **Hero** : Headline 6-8 mots + sous-titre benefice + CTA principal + visuel produit/hero
2. **Social proof bar** : Logos clients OU chiffres cles (X clients, X etoiles)
3. **Probleme** : 3 pain points avec icones — le lecteur se reconnait
4. **Solution** : 3 benefices concrets avec visuels — pas de features
5. **How it works** : 3 etapes simples (process visuel)
6. **Features** : Bento grid ou liste — details pour les convaincus
7. **Temoignages** : 3 minimum — photo + prenom + role + citation specifique
8. **Pricing** : 3 plans, highlight plan du milieu, FAQ inline
9. **FAQ** : 5-7 questions (objections frequentes)
10. **CTA final** : Section plein ecran, urgence ou garantie
11. **Footer** : Liens legaux + contact

### Regles headline
- 6-8 mots maximum
- Benefice direct (pas feature)
- Eviter les superlatifs vides ("meilleur", "revolutionnaire")
- Specifique > Generique ("Doublez votre taux de conversion" > "Ameliorez vos resultats")
- Tester au moins 3 angles: benefice direct / curiosite / transformation

### CTA — regles
- 1 seul CTA primaire par section
- Verbe action + benefice: "Commencer gratuitement", "Voir la demo", "Doubler mon trafic"
- Pas "Cliquer ici", "S'inscrire", "Soumettre"
- Couleur contrastee, jamais le meme gris que le fond
- Reduire friction: moins de champs = plus de conversions

## 7. A/B TESTING — PROTOCOLE

### Ce qu'on teste en priorite
| Element | Impact | Facilite | Priorite |
|---------|--------|---------|---------|
| Headline principal | Tres eleve | Facile | 1 |
| CTA (texte + couleur) | Eleve | Facile | 2 |
| Hero image/video | Eleve | Moyen | 3 |
| Pricing display | Eleve | Moyen | 4 |
| Ordre des sections | Moyen | Moyen | 5 |
| Nombre de champs form | Eleve | Facile | 6 |
| Social proof position | Moyen | Facile | 7 |
| Couleur fond hero | Bas | Facile | Pas prioritaire |

### Protocole A/B rigoureux
1. **Hypothese** : "Changer X devrait augmenter Y de Z% parce que..."
2. **Taille echantillon** : Calculer avec un calculateur A/B (Evan Miller) — min 95% confidence
3. **Une seule variable** par test — jamais tester 2 choses en meme temps
4. **Duree minimum** : 2 semaines OU seuil stat significatif atteint
5. **Segmenter** : Mobile vs Desktop, nouveaux vs recurrents
6. **Documenter** : Gagnant + % d'amelioration + date + hypothese

### Outils A/B testing
| Outil | Usage | Prix |
|-------|-------|------|
| PostHog | A/B testing + feature flags + analytics | Gratuit jusqu'a 1M events |
| Vercel Edge Config | Feature flags en edge | Gratuit |
| Split.io | Feature flags enterprise | Payant |
| Google Optimize | Defuncte depuis 2023 — eviter | -- |

### Erreurs frequentes
- Arreter le test trop tot (faux positif)
- Tester plusieurs elements en meme temps
- Ignorer la segmentation mobile/desktop
- Oublier les effets de saisonnalite (weekend vs semaine)
- Celebrer une victoire sans documenter pourquoi ca a marche

### Dashboard minimal viable
5 metriques a voir chaque matin:
1. **Signups hier** vs moyenne 7j
2. **Activations** (% trial → premier succes)
3. **MRR** + delta vs mois precedent
4. **Churn** du mois en cours
5. **Erreurs critiques** (Sentry)

### RGPD — conformite analytics
- Consentement avant tout tracking (banniere cookie)
- PostHog EU region (servers europe) — conforme RGPD
- Plausible = cookieless, pas de consentement requis
- Ne jamais stocker email en clair dans les events
- Droit a l'oubli: pouvoir supprimer un user de PostHog

## 9. EMAIL MARKETING — DELIVERABILITE

### Setup technique obligatoire
| Record DNS | Pourquoi | Impact |
|-----------|---------|--------|
| **SPF** | Autorise tes serveurs a envoyer | Anti-spam |
| **DKIM** | Signe cryptographiquement les emails | Anti-spoofing |
| **DMARC** | Politique si SPF/DKIM echouent | Reputation domaine |
| **BIMI** | Logo dans la boite mail | Credibilite |

### Regles de deliverabilite
- Echauffement du domaine : commencer a 50 emails/j, doubler chaque semaine
- Ratio HTML/texte : 60/40 minimum (eviter 100% HTML)
- Nettoyer la liste tous les 3 mois (supprimer inactifs 180j+)
- Bounce rate < 2% — nettoyer immediatement
- Spam rate < 0.1% — surveiller dans Google Postmaster
- Ne jamais acheter des listes — detruit la reputation du domaine

## 11. POSITIONNEMENT ET MESSAGING

### Framework de positionnement (April Dunford)
1. **Competiteurs alternatives** : Avec quoi les clients comparent
2. **Caracteristiques uniques** : Ce que tu as qu'ils n'ont pas
3**Valeur** : Benefice derive de ces caracteristiques
4. **Clients cibles** : Qui valorise le plus cette valeur
5. **Categorie de marche** : Le contexte qui rend ta valeur evidente

### Value Proposition Canvas
- **Jobs to be done** : Ce que le client essaie d'accomplir
- **Pains** : Ce qui empeche/frustre
- **Gains** : Ce qu'il espere obtenir

Ton produit doit connecter : Pain relievers + Gain creators → Products & services

### Messaging hierarchy
```
Tagline (6-8 mots) — Ce que tu fais
  ↓
Valeur principale (1 phrase) — Pour qui + transformation
  ↓
3 benefices cles (1 phrase chacun)
  ↓
Features et preuves (details)
```

## 12. ANTI-PATTERNS MARKETING

| Anti-pattern | Probleme | Solution |
|-------------|---------|---------|
| Parler features avant benefices | Personne ne s'identifie | "Tu obtiens X" pas "On a developpe Y" |
| CTA vague ("En savoir plus") | Pas d'action claire | CTA specifique avec benefice |
| Cibler tout le monde | Message dilue, CPM eleve | ICP strict, message niched |
| Ignorer le mobile | 60-70% du trafic est mobile | Mobile-first par defaut |
| Pas de social proof | Friction a l'achat | Minimum 3 temoignages avec photo |
| Changer de strategie chaque semaine | Impossible a mesurer | 3 mois minimum par canal |
| Optimiser avant d'avoir du volume | Pas de signaux fiables | Min 100 conversions avant d'optimiser |
| Copier la concurrence | Meme positionnement, invisible | Differentiation radicale sur 1 point |
| Email sans segmentation | Desabonnement eleve | Segmenter par comportement |
| Newsletter sans valeur | Desabonnement progressif | 80% valeur, 20% promo |
