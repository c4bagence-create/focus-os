## 1. GOOGLE ALGO 2024-2026

### Timeline des mises a jour majeures

**MARS 2024 — Core Update + Triple Spam Policy (45 jours)**
- Ciblage: contenu IA bas de gamme, backlinks manipulatifs, contenu non-original
- 3 nouvelles politiques spam:
  1. **Expired Domain Abuse**: domaines expires rachetes pour du contenu parasite → desindexation
  2. **Scaled Content Abuse**: production massive de contenu IA peu utile → penalite
  3. **Site Reputation Abuse**: parasite SEO sur sites autoritaires → desindexation des sections abusives

**AOUT 2024 — Core Update**
- Penalite renforcee contenu IA generique
- Boost des contenus "helpful" avec experience reelle auteur
- Priorite aux micro-niches ultra-specifiques vs mega-sites generiques

**MARS 2025 — AI Overviews Expansion**
- Google Answers (IA) occupe les positions 0-1 sur 60% des requetes informationnelles
- Requetes transactionnelles et comparatives toujours rentables en SEO classique
- Les sites avec E-E-A-T fort (Experience, Expertise, Authoritativeness, Trustworthiness) captent plus de featured snippets

### Ce qui fonctionne en 2026
- Contenu long (2000+ mots) avec experience personnelle reelle
- Backlinks contextuels depuis sites du meme secteur (pas des annuaires)
- Structure technique parfaite (Core Web Vitals, HTTPS, schema markup)
- E-E-A-T demontrable: pages auteur, mentions presse, profils sociaux actifs
- Mots-cles longue traine (< 500 recherches/mois) avec intent fort

| **AlsoAsked** | Questions Google "People Also Ask" | Freemium |

### Methode keyword research (de la niche au mot-cle)

**Etape 1 — Cartographie de la niche**
```
Niche principale → Sous-niches → Thematiques → Mots-cles
Ex: "finance personnelle" → "investissement" → "ETF" → "meilleur ETF monde 2026"
```

**Etape 2 — Criteres de selection**
- Volume: 100-10 000 recherches/mois (eviter les 0 et les mega-concurrents)
- KD (Keyword Difficulty): < 30 pour commencer, < 50 au bout de 6 mois
- Intent: Informatif (blog) / Commercial (comparatif) / Transactionnel (conversion)
- CPC: Plus le CPC est eleve, plus le mot-cle a valeur commerciale

**Etape 3 — Clustering thematique**
- Regrouper les mots-cles par intent et proximite semantique
- 1 cluster = 1 page pilier + articles satellites (maillage interne)
- Eviter le cannibalisme: 2 pages qui se font concurrence sur le meme keyword

### Formule longue traine qui convertit
```
[qualificatif] + [produit/service] + [localisation ou contexte] + [intent]
Ex: "meilleur logiciel comptabilite TPE France 2026"
```

## 3. ON-PAGE SEO

### Structure d'un article qui classe

```html
Title tag: [Mot-cle principal] — [Benefice] | [Marque] (50-60 chars)
Meta desc: [Prompt + mot-cle] + [CTA] (150-160 chars)

H1: [Mot-cle principal] (1 seul H1)
  Introduction: 150-200 mots, mot-cle dans la 1ere phrase

H2: [Sous-thematique 1]
  Contenu, mot-cle LSI

H2: [FAQ — Les questions que les gens posent]
  Format: Question + Reponse courte (pour featured snippet)

Schema markup: Article + FAQ + BreadcrumbList
```

### Core Web Vitals (seuils 2026)
| Metrique | Objectif | Critique |
|---------|---------|---------|
| LCP (chargement) | < 2.5s | > 4s = penalite |
| FID/INP (interactivite) | < 200ms | > 500ms = penalite |
| CLS (stabilite visuelle) | < 0.1 | > 0.25 = penalite |

### Schema Markup prioritaires
```json
// Pour les articles/blogs
{ "@type": "Article", "author": {"@type": "Person"}, "datePublished": "..." }

// Pour les FAQ
{ "@type": "FAQPage", "mainEntity": [{"@type": "Question", "name": "...", "acceptedAnswer": {...}}] }

// Pour les produits/comparatifs
{ "@type": "Product", "aggregateRating": {...} }

// Pour les recettes/howto
{ "@type": "HowTo", "step": [...] }
```

## 4. BACKLINKS STRATEGY

### Pyramide de backlinks (du moins risque au plus risque)

**Tier 1 — Fondation (safe, toujours)**
- Guest posts sur sites de la meme niche (DA 30-70)
- Mentions presse (PR, interviews, etudes citees)
- Partenariats editoriaux (echanger des articles)
- HARO/Connectively (journalistes cherchent des experts)

**Tier 2 — Acceleration (modere)**
- Digital PR: etudes originales, data, infographies (les gens linkent naturellement)
- Broken link building: trouver les liens morts chez les concurrents, proposer sa page
- Resource pages: pages "liens utiles" dans ta niche — demander l'ajout
- Podcast appearances: backlink depuis la page episode

**Tier 3 — Parasite SEO (avance, a maitriser)**
- Publier sur des sites a fort DA qui autorisent le contenu externe (Medium, Substack, LinkedIn Pulse)
- Forum profiles sur Reddit, Quora avec liens contextuels (pas de spam)
- Press release sur des aggregateurs (PRNewswire, BusinessWire)

**A EVITER (risque de penalite)**
- Achat de liens en masse (PBN)
- Echanges de liens reciproques a grande echelle
- Footer links / sidebar links sur des sites non-thematiques
- Liens depuis des fermes de contenu IA

### Outreach email template (approche qui convertit)

```
Objet: [Nom du site] — lien casse sur votre page [titre]

Bonjour [Prenom],

En lisant votre article "[titre]", j'ai remarque que le lien vers
[ancre du lien mort] est casse (erreur 404).

J'ai ecrit un article complet sur le sujet qui pourrait remplacer
cette reference: [URL de ton article]

[1 phrase sur pourquoi ton contenu est meilleur/plus a jour]

Bonne continuation,
[Signature]
```

Taux de reponse moyen: 5-15%. Envoyer a 50-100 sites pour obtenir 5-10 liens.

## 5. CONTENT STRATEGY

### Framework TOFU-MOFU-BOFU

```
TOFU (Top of Funnel) — Informatif, volume eleve
→ "comment fonctionne X", "qu'est-ce que Y", tutoriels
→ Objectif: trafic + notoriete

MOFU (Middle of Funnel) — Comparatif, intent commercial
→ "meilleur X", "X vs Y", "alternatives a Z"
→ Objectif: consideration + lead

BOFU (Bottom of Funnel) — Transactionnel, intent achat
→ "prix X", "acheter Y", "[produit] avis"
→ Objectif: conversion
```

### Calendrier editorial efficace
- 1-2 articles TOFU par semaine (trafic de masse)
- 2-4 articles MOFU par mois (lead generation)
- 1 article BOFU par mois (conversion directe)
- Mise a jour des anciens articles tous les 6 mois (signal de fraicheur)

### Regle E-E-A-T pour le contenu
1. **Experience**: Partager son experience reelle (screenshots, resultats, cas personnels)
2. **Expertise**: Citer des sources, montrer sa maitrise technique du sujet
3. **Autorite**: Etre cite ailleurs, avoir des backlinks de qualite
4. **Confiance**: Pages legales, auteur identifie, mentions de presse, avis

## 6. AUDIT SEO — CHECKLIST

### Technique
- [ ] HTTPS actif, pas de mixed content
- [ ] Sitemap XML soumis a Google Search Console
- [ ] Robots.txt correct (pas de blocage des pages importantes)
- [ ] Core Web Vitals dans le vert
- [ ] Pas de contenu duplique (canonical tags)
- [ ] Balises canoniques sur les pages paginées
- [ ] Schema markup sur les pages cles

### Contenu
- [ ] 1 mot-cle principal par page (pas de cannibalisme)
- [ ] H1 unique et contenant le mot-cle
- [ ] Meta title et description uniques sur chaque page
- [ ] Images avec alt text descriptif
- [ ] Liens internes vers les pages importantes (maillage)
- [ ] Pas de "thin content" (pages < 300 mots sans valeur ajoutee)

### Backlinks
- [ ] Profil de backlinks verifie (pas de liens toxiques)
- [ ] Anchor text diversifie (pas 90% de texte exact)
- [ ] Desaveu des liens toxiques si necesssaire (Google Disavow Tool)
